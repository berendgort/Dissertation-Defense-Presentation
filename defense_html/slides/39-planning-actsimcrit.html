Planning Agent · the ActSimCrit loop

plan · simulate · critique · repeat until intent is met
five-phase loop · 5 LLM calls + 1 sim per iteration
four mechanisms that make the loop work
i · ACTION BATCHING one LLM call → k actions
Group logically connected steps into one call instead of replanning each action.
migrate + migrate + low-power
ii · THREE-TIER STATE baseline · accepted · simulated
Atomic promotion — only validated batches update accepted; failed ones leave it untouched.
baseline → accepted → simulated
iii · REJECTION-FEEDBACK structured reason → next try
Each failed batch records why it was rejected; the schema excludes it from the next selection.
rejected: node_5 over capacity (+0.4 core)
next: try node_7 instead
iv · DEADLOCK DETECTION iter & reject thresholds
Infeasible intents terminate cleanly — propose an achievable alternative instead of retrying forever.
iter cap & reject cap → alt-intent
Output is a validated plan — not a guess. The infra agent executes, it does not diagnose.
ActSimCrit · plan → simulate → critique → accept · validated plan out
