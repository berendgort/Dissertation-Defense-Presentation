Problem 3 · Distributed orchestration ecosystems
How do many agents coordinate over one physical infrastructure?
Research directions
R1 · CONFLICT-DETECTION PROTOCOLS
Lightweight consensus / lock-lease schemes for intended actions; detect conflicts before actuation, not after. Distributed-systems classics applied to probabilistic planners.
R2 · MODEL-PORTABILITY STANDARDS
A canonical intent schema (semantic, not syntactic) so swapping provider X ⇒ Y preserves agent behaviour: an OpenAPI-equivalent for agent tool-calls.
R3 · FEDERATED AGENTIC OPERATIONS
Cross-operator federation: sharing intent, not state. Enables multi-administrative-domain orchestration (edge + public cloud + private DC) without surrendering control.
Conflict protocols · intent schemas · federated operation · systems research, not tool swaps
