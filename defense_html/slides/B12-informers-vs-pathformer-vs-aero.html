Informers vs Pathformer vs AERO: where does each fit?

INFORMER FAMILY
Long-horizon forecasting transformers
~10–30 M params
Appeared in the magazine J4 as part of the attention-mechanism survey, not a baseline for the AERO core result. Too heavy for edge.
heavy · cloud-only
PATHFORMER
SotA multi-scale forecaster (2024)
~1.2 M params
The main AERO baseline. Chosen because it represents the current peak of small-ish accurate forecasting. Does not fit far-edge memory budgets; that is the narrative in the AERO SotA and efficiency-result slides.
overshoots edge budget
AERO (ours)
Adaptive-periodicity attention
599 params · 0.38 ms
3000+× fewer params than Pathformer. Achieves competitive MAE because the architecture learns the per-service periodicity instead of carrying it in parameters.
fits far-edge budget
TAKEAWAY
AERO's headline is not beating Pathformer on MAE; it is matching Pathformer at 3000× smaller, which enables far-edge deployment.
Informer · Pathformer · AERO · roles, not rankings
