EXPERIMENT 1 · CONSTRAINT RESOLUTION · S1 · S3
S1
Full Node
Deploy a 2-core service onto a 100% utilized node. The agent must detect the conflict, migrate existing services to create capacity, then deploy.
Tests: basic constraint reasoning + migration planning
S2
Low-Power Conflict
Scale API to 1.4 cores while putting edge-1 in low-power mode simultaneously. Services cannot run on low-power nodes · agent must migrate first.
Tests: dependency detection + multi-step coordination
S3
Compound Constraint
Scale API to 2.4 cores + power down edge-1 · migrating both services creates a 0.52-core deficit. Intent is infeasible as stated.
Tests: deadlock detection + creative redistribution
How we measure success
For each scenario we pre-define all valid end-states that satisfy the goal. A run counts as successful if the agent reaches any of those states.
