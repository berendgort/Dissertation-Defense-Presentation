Design rationale

Edge efficiency comes from exploiting signal structure

Three pain points force three decisions; each shrinks the model and the inference cost.
Pain point
Edge parameter budget
< 1 K params leaves almost zero expressive capacity under standard architectures
Workload drift
dominant period shifts between training and deployment; static period assumptions break
Inference latency
any attention / KV-cache already exceeds the 50 ms decision cycle on edge CPUs
Design decision
Small head on period-aligned features
reshape the trace by its dominant period → a tiny MLP suffices
Adaptive periodicity detection
τ re-estimated online every window; follows drift automatically
Linear ops only · no attention, no KV-cache
→ 0.38 ms inference on commodity edge CPU
AERO · adaptive periodicity + linear head · 0.38 ms on edge CPU
