Design rationale

Edge efficiency comes from exploiting signal structure

Pain point
Edge parameter budget
models that fit in <1K params have near-zero expressive capacity under standard architectures
Workload drift
distribution shifts between training and deployment · static period assumptions break
Inference latency
even small transformers with KV-cache hit >1ms on edge CPUs
Design decision
Small head on period-aligned features
Adaptive periodicity detection
re-estimated online each window
No attention, no KV-cache
linear ops only · 0.38ms on edge CPU is achievable
AERO · rationale · adaptive periodicity · no attention

Mechanism
14/56   AERO
AERO architecture · three modules

Adaptive periodicity in three steps

01
Period Detection
Find the dominant repeat cycle in the workload trace
Uses autocorrelation on the input trace · re-estimated every window · adapts to drift automatically
02
Adaptive Forecasting
Reshape input around τ · forecast with a lightweight SparseTSF head
Period alignment reduces capacity needed · no attention, no KV-cache · 599 params · 0.38ms
03
Layer Caching
Cache layers optimized for each period · reuse when the same pattern recurs
Same workload pattern recurs → reuse cached layer · reduces compute overhead significantly
AERO · period detection → adaptive forecasting → layer caching
