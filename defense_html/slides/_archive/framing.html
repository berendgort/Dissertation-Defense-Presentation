04/56   Framing
The goal

Zero-touch: operator sets intent, not actions

Operator sets intent. The loop does the rest.
Goals, SLAs, constraints in · orchestration decisions out. Operator reviews exceptions only.
Four problems · one per stage of the loop
P1 · Predict
Forecasts don't run where the service runs
accurate models too big · small models lose accuracy
P2 · Predict
A new model must be refit per service
O(N) retraining burden · doesn't scale
P3 · Decide
No intelligent action from intent
no component today translates natural-language goals into orchestration plans
P4 · Observe
Operator analyzes the stack by hand
hours of dashboards, guesswork, tribal knowledge between intent and action
Zero-touch loop · four open problems, one per stage

The intelligent-action gap
  • Natural-language intent → rules out heuristics, reinforcement learning (RL), mixed-integer linear programming (MILP)
  • No component reads intent → plan
  • Operator fills the gap manually
  • Scales with attention, not workload
